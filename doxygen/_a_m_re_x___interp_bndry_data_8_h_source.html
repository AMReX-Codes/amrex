15 /*
16  An InterpBndryData object adds to a BndryData object the ability to
17  manipulate and set the data stored in the boundary cells.
18 
19  The Interpbndrydata class is a virtual base class derived from
20  BndryData. It is intended to provide a more physical method for
21  filling boundary-related data. Boundary values in a BndryData object
22  are stored in FabSets around each grid in the domain, and the
23  InterpBndryData class provides a mechanism for filling these FabSets,
24  consistent with AMR-like numerical discretizations. When asked to
25  set its boundary values, an InterpBndryData object:
26 
27  Fills with physical boundary values if the FAB is on the
28  domain boundary - the corresponding values are presumed to be
29  stored in the ghost cells of a MultiFab given to the boundary filling
30  routine
31 
32  Fills on intersection with data from the VALID region of the
33  input MultiFab, ensuring that adjacent FABs use consistent data at
34  their intersection, and otherwise,
35 
36  Fills with values interpolated from a coarser FAB that
37  bounds the cells that do not meet the above two criteria
38 */
